# API Authentication Guide

## Overview

CampaignEngine API endpoints are secured using **API key authentication**.
Every external caller must include a valid API key in the `X-Api-Key` HTTP header.

The system also enforces:
- **HTTPS** for all requests (HTTP connections are redirected automatically).
- **Per-key rate limiting** (default 1000 requests/minute, configurable per key).
- **Key expiry** (default 1 year, configurable per key).

---

## Authentication Method

### API Key Header

Include the API key in every request:

```
X-Api-Key: ce_aBcDeFgH...
```

Example using curl:

```bash
curl -X POST https://your-server/api/send \
  -H "X-Api-Key: ce_aBcDeFgH..." \
  -H "Content-Type: application/json" \
  -d '{
    "templateId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "channel": "Email",
    "recipient": { "email": "user@example.com" },
    "data": { "firstName": "Alice" }
  }'
```

Example using .NET `HttpClient`:

```csharp
using var client = new HttpClient();
client.DefaultRequestHeaders.Add("X-Api-Key", "ce_aBcDeFgH...");

var payload = new { templateId = "...", channel = "Email", ... };
var response = await client.PostAsJsonAsync("https://your-server/api/send", payload);
```

## API Key Format

API keys are generated by CampaignEngine and follow this format:

```
ce_<32 URL-safe Base64 characters>
```

Example: `ce_aBcDeFgHiJkLmNoPqRsTuVwXyZ01234`

Key properties:
- **Length**: 35 characters total.
- **Prefix**: Always starts with `ce_` to distinguish CampaignEngine keys from other secrets.
- **Entropy**: 192 bits of cryptographic randomness (24 random bytes encoded as Base64).
- **Storage**: The plaintext key is **never stored** — only a BCrypt hash is persisted. The plaintext is shown once at creation and cannot be retrieved again.

## Managing API Keys

API keys are managed by **Administrators only** via the Admin UI or the REST API.

### Admin UI

Navigate to **Admin > API Keys** (`/Admin/ApiKeys`):

- **Create**: click "Create API Key", enter a name and optional settings.
- **Revoke**: click "Revoke" on any active key. Revocation is immediate and permanent.
- **Rotate**: click "Rotate" to generate a new key while revoking the old one atomically.

The plaintext key value is shown **only once** immediately after creation or rotation.
Copy it to a secrets manager before closing the page.

### REST API

All endpoints require `Admin` role.

#### List all keys

```
GET /api/apikeys
Authorization: (admin cookie or API key with Admin role)
```

Response:

```json
[
  {
    "id": "...",
    "name": "CRM Integration",
    "keyPrefix": "ce_aBcDe",
    "isActive": true,
    "isExpired": false,
    "expiresAt": "2027-03-20T00:00:00Z",
    "lastUsedAt": "2026-03-20T14:30:00Z",
    "createdBy": "admin@example.com",
    "rateLimitPerMinute": null,
    "description": "Used by CRM to trigger transactional emails"
  }
]
```

#### Create a key

```
POST /api/apikeys
Content-Type: application/json

{
  "name": "CRM Integration",
  "description": "Optional notes about the consumer",
  "expiresInDays": 365,
  "rateLimitPerMinute": 500
}
```

Response (`201 Created`):

```json
{
  "key": { "id": "...", "name": "CRM Integration", ... },
  "plaintextKey": "ce_aBcDeFgHiJkLmNoPqRsTuVwXyZ01234"
}
```

The `plaintextKey` field is **only present in the creation response**. Store it securely.

#### Revoke a key

```
POST /api/apikeys/{id}/revoke
```

Response: `204 No Content`. The key stops working immediately.

#### Rotate a key

```
POST /api/apikeys/{id}/rotate
```

Response (`201 Created`): same format as create. The old key is revoked atomically; the new plaintext key is returned once.

## Error Responses

### 401 Unauthorized

Returned when:
- The `X-Api-Key` header is missing on an endpoint that requires authentication.
- The key is invalid (wrong value), expired, or revoked.

```json
{
  "status": 401,
  "title": "Unauthorized",
  "detail": "The provided API key is invalid, expired, or has been revoked."
}
```

### 429 Too Many Requests

Returned when the per-key rate limit is exceeded.

```json
{
  "status": 429,
  "title": "Too Many Requests",
  "detail": "Rate limit exceeded. Maximum 1000 requests per minute per API key.",
  "retryAfterSeconds": 45
}
```

The response includes a `Retry-After` header with the number of seconds until the window resets.

---

## HTTPS Enforcement

All CampaignEngine API endpoints require HTTPS.

- HTTP requests are automatically redirected to HTTPS (`301 Moved Permanently`).
- HSTS (`Strict-Transport-Security`) is enabled in production environments to prevent downgrade attacks.
- Do not disable `RequireHttps` in production (`ApiKey.RequireHttps` must remain `true`).

---

## Rate Limiting

Rate limiting is enforced per API key using a **fixed window algorithm** (window = 1 minute).

| Setting | Default | Location |
|---|---|---|
| Requests per minute | 1000 | `ApiKey.DefaultRateLimitPerMinute` in `appsettings.json` |
| Per-key override | null (use default) | `RateLimitPerMinute` field on each key |

To change the system-wide default:

```json
"ApiKey": {
  "DefaultRateLimitPerMinute": 2000
}
```

To set a per-key override, pass `rateLimitPerMinute` when creating the key.

When a key reaches its limit, subsequent requests receive `429 Too Many Requests` with a `Retry-After` header. The window resets automatically after 60 seconds.

---

## Key Expiry

Keys expire after a configurable number of days (default: 365 days from creation).

| Setting | Default | Notes |
|---|---|---|
| `ExpiresInDays` (per key) | 365 | Set to `null` for a non-expiring key |
| System default | 365 | Configurable via `ApiKey.DefaultExpiresInDays` |

Expired keys return `401 Unauthorized`. To restore access, create or rotate a key.

---

## Security Best Practices

1. **Store keys in a secrets manager** — never hard-code them in source code or configuration files committed to version control.
2. **Rotate keys regularly** — use the rotate operation to issue a new key without downtime.
3. **Revoke keys immediately** if compromised — revocation is instantaneous.
4. **Use descriptive names** — name keys after the consuming system (e.g., "CRM Integration", "Mobile App Backend") so audit logs are readable.
5. **Set appropriate rate limits** — use per-key overrides to restrict high-risk or low-priority consumers below the system default.
6. **Prefer short expiry for sensitive integrations** — shorter `ExpiresInDays` reduces the blast radius of a leaked key.

---

## Configuration Reference

`appsettings.json` section `ApiKey`:

```json
"ApiKey": {
  "DefaultExpiresInDays": 365,
  "DefaultRateLimitPerMinute": 1000,
  "RequireHttps": true
}
```

| Property | Type | Default | Description |
|---|---|---|---|
| `DefaultExpiresInDays` | int | 365 | Key lifetime in days when not specified at creation |
| `DefaultRateLimitPerMinute` | int | 1000 | System-wide rate limit applied when the key has no per-key override |
| `RequireHttps` | bool | true | Enforces HTTPS for API requests. **Set to `false` only in development.** |

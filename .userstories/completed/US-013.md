## Implementation Summary
`DocxRunMerger` (in `Infrastructure/Rendering/`) exposes a single public method `MergeRuns(WordprocessingDocument doc)` that visits every paragraph in the main document body, all registered HeaderParts, and all registered FooterParts. Within each paragraph, adjacent `<w:r>` runs whose `<w:rPr>` serialises identically (or both absent) are merged into a single run by concatenating their `<w:t>` text nodes. Bookmark elements (`<w:bookmarkStart>`, `<w:bookmarkEnd>`) encountered between compatible runs are collected during the merge pass and re-emitted immediately after the merged run, preserving them as paragraph-level siblings. Smart quotes (U+201C / U+201D) are normalised to ASCII U+0022 on every text node during the flush step. 22 unit tests (all passing) cover fragmented placeholder merge, bookmark preservation, header/footer traversal, and smart-quote normalisation variants.

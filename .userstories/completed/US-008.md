## Implementation Summary
Added `GET /api/templates/{id}/docx` to `TemplatesController` protected by `RequireAuthenticated` policy (covers Designer, Admin, and Operator/CampaignManager — all three roles). The endpoint delegates to a new `ITemplateService.GetDocxBodyAsync(id)` method implemented in `TemplateService`. The service throws `NotFoundException("Template", id)` when the template does not exist, and `DomainException` when the channel is not Letter or no DOCX is attached; these propagate to the `GlobalExceptionMiddleware` which maps them to HTTP 404 and 422 respectively. The response streams the DOCX bytes with MIME type `application/vnd.openxmlformats-officedocument.wordprocessingml.document` and `Content-Disposition: attachment; filename="{templateName}.docx"`. Authorization note: the codebase uses "Operator" for the campaign manager persona; `RequireAuthenticated` covers all three roles (Designer, Operator, Admin), which matches the F-110 specification.

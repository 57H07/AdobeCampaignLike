## Implementation Summary
The LetterDispatcher was completely rewritten to accept pre-rendered DOCX bytes via DispatchRequest.BinaryContent and write one DOCX file per recipient through PrintProviderFileDropHandler.WriteFileAsync. The new file naming convention is {campaignId}_{recipientId}_{timestamp}.docx. All legacy PDF consolidation infrastructure (DinkToPdf, PdfSharp, LetterPostProcessor, PdfConsolidationService, FlushBatchAsync, LetterManifestGenerator) was removed from both source and test projects. The LetterDispatcher constructor now takes only PrintProviderFileDropHandler and LetterOptions (no post-processor registry or consolidation service). I/O failures propagate as LetterDispatchException(isTransient: true) to enable Hangfire retries.

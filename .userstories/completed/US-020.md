## Implementation Summary

Created `IDocxTemplateRenderer` in the Application layer and `DocxTemplateRenderer` in Infrastructure, which orchestrates the full DOCX rendering pipeline: `DocxRunMerger` → `DocxConditionalBlockRenderer` → `DocxTableCollectionRenderer` → `DocxPlaceholderReplacer`. Added `PreviewDocxAsync(id, DocxPreviewRequest)` to `ITemplatePreviewService` and implemented it in `TemplatePreviewService`, loading DOCX bytes from `ITemplateBodyStore` and running the full pipeline ephemerally. The `POST /api/templates/{id}/preview/docx` endpoint returns a `FileContentResult` with MIME type `application/vnd.openxmlformats-officedocument.wordprocessingml.document` and `Content-Disposition: attachment; filename="preview-{sanitisedName}.docx"`. All rendering components are registered as singletons (stateless). 6 API integration tests cover success path, header validation, filename sanitisation, 404, and 422 error cases.

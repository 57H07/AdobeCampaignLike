## Implementation Summary

Swashbuckle.AspNetCore was already installed but minimally configured. The implementation adds full OpenAPI 3.0 configuration via a dedicated `SwaggerServiceExtensions` class: X-Api-Key security scheme, XML documentation from both Web and Application assemblies, request/response example providers (via Swashbuckle.AspNetCore.Filters), and a custom CSS theme for the Swagger UI. The UI is available at `/swagger` in Development and Staging environments only (business rule BR-1); the raw JSON spec at `/swagger/v1/swagger.json` is available in all environments. Seven integration tests using WebApplicationFactory verify spec validity, required endpoints, and security scheme presence.

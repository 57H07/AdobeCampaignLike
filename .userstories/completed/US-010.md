## Implementation Summary
Two new multipart upload endpoints were added to `TemplatesController`: `POST /api/templates/letter` and `PUT /api/templates/{id}/letter`, both decorated with `[RequestSizeLimit(10_485_760)]` to enforce the 10 MB Kestrel limit. Both endpoints perform a second controller-level size check and return HTTP 413 with a clear error message if the file exceeds the limit. `CreateTemplateRequest` received a new optional `FileSizeBytes` property, and `TemplateService.CreateAsync` re-validates this value against the 10 MB threshold (defense-in-depth per F-204), throwing `ValidationException` if exceeded. Six unit tests in `FileSizeLimitTests` cover: exactly-at-limit success, one-byte-under success, null size (skip validation), 11 MB rejection, one-byte-over rejection, and error message content validation.

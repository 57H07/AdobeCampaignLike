### Notes
- The TemplatePreviewService follows the existing pattern: loads template from DbContext, resolves sub-templates recursively (ISubTemplateResolverService), fetches up to 5 rows via IDataSourceConnector (read-only, no filters), renders with ITemplateRenderer (Scriban), applies channel post-processing via IChannelPostProcessorRegistry.
- Missing placeholder keys are detected by comparing IPlaceholderParserService.ExtractPlaceholders() output against the sample row's field names; missing keys are rendered as empty string (no render failure) and reported back in the result.
- For Letter channel, the PDF bytes are returned as Base64 in the result; the Web UI provides a data-URI download link (no server-side file storage needed for preview).
- The preview modal uses a sandboxed iframe (`sandbox="allow-same-origin"`) for HTML output to prevent script execution in the preview frame.
- All 622 tests pass (14 new + 608 pre-existing).

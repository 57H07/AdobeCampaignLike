## Implementation Summary

The CampaignAttachment entity and EF Core configuration were pre-existing. The implementation added the full service stack: AttachmentValidationService enforces the extension whitelist (.pdf, .docx, .xlsx, .png, .jpg) and size limits (10 MB per file, 25 MB total). FileUploadService writes files to a configurable UNC/local path under campaign-scoped subdirectories with GUID-prefixed names. DynamicAttachmentResolver reads per-recipient file paths from data source fields at send time, logging a warning and skipping missing files without blocking delivery (BR-5). The REST API exposes GET, POST (multipart), POST (dynamic mapping), and DELETE endpoints under /api/campaigns/{id}/attachments. The Campaign Detail Razor Page gained an attachment panel with a file uploader and a dynamic field mapper backed by fetch calls to the API. 45 tests cover validation logic and all missing-file scenarios.

## Implementation Summary

`DocxFilePathHelper.Build(templateId, version)` generates relative paths following the strict convention `templates/{templateId}/v{version}.docx`. Both `CreateTemplateRequest` and `UpdateTemplateRequest` gained an optional `DocxContent` (Stream) property. When `DocxContent` is provided, `TemplateService` generates the versioned path, calls `ITemplateBodyStore.WriteAsync`, and stores the relative path in `BodyPath`. Directory auto-creation is handled by `FileSystemTemplateBodyStore.WriteAsync` (from US-002) which calls `Directory.CreateDirectory` before writing. When `DocxContent` is null (non-Letter channels), the caller-supplied `BodyPath` is used unchanged.

`TemplateService` now requires `ITemplateBodyStore` in its constructor (already registered in DI via `ServiceCollectionExtensions.AddInfrastructure()`). All 5 existing test files that manually instantiated `TemplateService` were updated to pass a mock `ITemplateBodyStore`.

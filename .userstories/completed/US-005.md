## Implementation Summary
`TemplateService.CreateAsync` now uses an atomic write pattern: the file (DOCX or HTML) is written to disk first inside a try-catch block, then the DB transaction commits. If the DB commit fails for any reason, the orphaned file is deleted synchronously via `ITemplateBodyStore.DeleteAsync`. `TemplateService.UpdateAsync` follows the same pattern plus copies the previous body file to a history path (e.g. `history/v{n}.docx`) before writing the new version. `DbUpdateConcurrencyException` from EF Core is caught and re-thrown as `ConcurrencyException`, which `GlobalExceptionMiddleware` translates to HTTP 409 Conflict. Unit tests cover the create success path, DB failure file cleanup, and concurrent update 409 response; an integration test validates the history copy behaviour.

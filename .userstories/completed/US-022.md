## Implementation Summary
Added `BinaryContent` (`byte[]?`) property to `DispatchRequest` alongside the now-nullable `Content` (`string?`) property. Full mutual-exclusivity semantics documented in XML doc comments on the class and both properties: Letter channel sets `BinaryContent` and leaves `Content` null; Email/SMS channels set `Content` and leave `BinaryContent` null. Downstream dispatchers determine the dispatch path via null checks. Minor null-safety fixes applied to `EmailDispatcher` (null-coalescing fallback on `Content`) and `SmsDispatcher` (local variable for `content ?? string.Empty`) so existing Email/SMS dispatch behaviour is preserved without compiler warnings. 10 unit tests cover default nulls, property assignment, channel-specific mutual exclusivity patterns, and the dispatcher null-check routing logic.

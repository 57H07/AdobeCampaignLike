## Implementation Summary

`RestApiConnector` implements `IDataSourceConnector` using `IHttpClientFactory`. It parses a semicolon-delimited connection string (via `RestApiConnectionString`) to configure the endpoint URL, authentication mode (None/ApiKey/Bearer/OAuth2), pagination strategy (none/page-param/Link-header), and a dot-notation JSON data path for nested responses. Authentication uses `RestApiOAuth2TokenCache` (Singleton) for client-credentials token caching. Filtering is applied in-memory after all pages are fetched. Business rules enforced: 60s timeout, 3-attempt exponential backoff retry, 50MB response size limit via `LimitedStream`. `DataSourceConnectorRegistry` (implementing the new `IDataSourceConnectorRegistry` interface) resolves connectors by `DataSourceType` at runtime; all services consuming data sources were updated to use this registry instead of injecting `IDataSourceConnector` directly.

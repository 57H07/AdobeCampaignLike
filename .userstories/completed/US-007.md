## Implementation Summary
`CreateTemplateRequest` and `UpdateTemplateRequest` now accept an optional `HtmlContent` stream. When provided, `TemplateService` writes the HTML to `ITemplateBodyStore` using the `HtmlFilePathHelper.Build(templateId, version)` convention (`templates/{id}/v{version}.html`) — exactly parallel to the DOCX flow from US-006. A new extension method `TemplateBodyStoreExtensions.ReadAllTextAsync` reads a stored file as a UTF-8 string; it is an extension method (not a default interface method) to ensure Moq mock setups on `ReadAsync` are honoured. All callers that previously used `template.BodyPath` as a raw string template now load the HTML body via `ITemplateBodyStore` before rendering or validation: `SingleSendService`, `TemplatePreviewService`, `SubTemplateResolverService`, `TemplatesController` (placeholder endpoints), and the `Manifest.cshtml.cs` Razor Page. `PublishAsync` was restructured to check domain status before body-store operations so that `DomainException` propagates cleanly for non-Draft templates without requiring manifest mocks.

## Implementation Summary
Replaced the `HtmlBody` database column on `Template` and `TemplateHistory` with a file-based
storage model. The `Template` entity now has `BodyPath` (nvarchar(500), required), `BodyChecksum`
(nvarchar(64), nullable SHA-256 hex string), and `RowVersion` (rowversion concurrency token).
`TemplateHistory` has `BodyPath` and `BodyChecksum` only (no concurrency token needed on history).
The EF Core migration `20260329165000_AddBodyPathAndChecksumToTemplates` drops the old columns and
adds the new ones. All downstream consumers (services, DTOs, controllers, Razor Pages, tests) were
updated from `HtmlBody` to `BodyPath`/`BodyChecksum`. The deployment note documents that rollback
requires a database restore from backup (no down-migration strategy).

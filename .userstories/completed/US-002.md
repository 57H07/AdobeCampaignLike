## Implementation Summary
`FileSystemTemplateBodyStore` implements `ITemplateBodyStore` and is located in `Infrastructure/Storage/`. It resolves all paths relative to a configurable `RootPath` (default: `"templates"`), bound from `appsettings.json` section `CampaignEngine:Templates:Storage` via `TemplateBodyStoreOptions`. `WriteAsync` uses the atomic write pattern: stream is copied to a `.tmp` file with `FileOptions.Asynchronous`, then renamed with `File.Move(overwrite: true)`. `ReadAsync` returns a `FileStream`; it throws `TemplateBodyNotFoundException` for null/missing paths and `TemplateBodyCorruptedException` for any other I/O failure. `DeleteAsync` is a no-op for missing files. The store is registered as `Singleton<ITemplateBodyStore>` in `ServiceCollectionExtensions`. 22 unit tests cover all happy-path and exception scenarios.

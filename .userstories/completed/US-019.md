## Implementation Summary

Two pure static function classes were created in `Application/Rendering/CustomFunctions/`: `FormatDateFunction.Execute(value, format)` and `FormatCurrencyFunction.Execute(value, symbol)`, with no Scriban dependency (Application-layer compatible). The `TemplateCustomFunctions` ScriptObject in Infrastructure was updated to delegate to these classes, keeping the Scriban adapter in Infrastructure and the pure logic in Application. Registration is done at render time via `ScribanTemplateRenderer.BuildScribanContext()`, which pushes `TemplateCustomFunctions.CreateAndRegister()` into the Scriban context before data values. Both functions are available in Email and SMS templates as `format_date` and `format_currency`; their unavailability in DOCX/Letter templates is documented in `advanced-rendering-syntax.md` and `template-syntax-reference.md`. 38 unit tests cover multiple date formats, currency symbols, null handling, and invariant-culture output.

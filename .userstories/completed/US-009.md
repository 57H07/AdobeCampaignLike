## Implementation Summary
`DocxValidationService` implements five sequential validation gates per F-203: (1) case-insensitive .docx extension check with explicit .docm rejection; (2) valid ZIP archive; (3) [Content_Types].xml presence; (4) WordprocessingDocument.Open with MainDocumentPart check; (5) vbaProject.bin absence. All failures throw `ValidationException` (Domain layer) which the global exception middleware maps to HTTP 422. The service is registered as scoped via `IDocxValidationService` in `AddApplication()`. 22 unit tests cover all rules using programmatic in-memory ZIP fixtures (no binary files committed).

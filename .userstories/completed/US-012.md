### Notes:

- Table, list, and conditional rendering are native Scriban capabilities — no custom parser needed. The `{{ for row in collection }}...{{ end }}` and `{{ if condition }}...{{ end }}` syntax was already supported by `ScribanTemplateRenderer` from US-011.
- TASK-012-04 (custom functions) required implementing `TemplateCustomFunctions` as a `ScriptObject` subclass. Scriban requires functions to be methods on a `ScriptObject` instance; raw `Func<>` delegates are not supported as values via `SetValue`.
- `format_currency` uses `CultureInfo.InvariantCulture` (`N2` format) to produce consistent output (`1,234.56`) regardless of the server's OS locale setting.

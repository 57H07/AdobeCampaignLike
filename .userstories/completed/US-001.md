## Implementation Summary
Defined the `ITemplateBodyStore` interface in `CampaignEngine.Application.Interfaces.Storage` with three async methods: `ReadAsync(string path, CancellationToken ct)` returning `Task<Stream>`, `WriteAsync(string path, Stream content, CancellationToken ct)` returning `Task<string>` (the resolved path), and `DeleteAsync(string path, CancellationToken ct)`. Two exception types were created in `CampaignEngine.Application.Interfaces.Exceptions`: `TemplateBodyNotFoundException` (for missing or null/empty paths) and `TemplateBodyCorruptedException` (for files that exist but cannot be opened or parsed); both carry a `Path` property. All members have full XML documentation including exception contracts on `ReadAsync`. The interface carries no Infrastructure dependencies — implementations (US-002) will live in `CampaignEngine.Infrastructure` and throw these Application-layer exceptions.

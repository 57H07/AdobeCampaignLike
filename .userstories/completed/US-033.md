## Implementation Summary

The rate limiting stack now uses `IApiKeyRateLimiter` (Application interface) backed by `ApiKeyRateLimiter` (Infrastructure, Singleton) which implements a sliding 1-minute window per API key using a timestamp queue with per-key locking. The `ApiKeyRateLimitingMiddleware` was rewritten to delegate to this service and now emits `X-RateLimit-Limit`, `X-RateLimit-Remaining`, and `X-RateLimit-Reset` headers on every API-key-authenticated response, plus `Retry-After` on 429 responses. A new `PATCH /api/apikeys/{id}/rate-limit` endpoint and a Razor Pages `EditRateLimit` page allow runtime rate limit updates. A `GET /api/apikeys/rate-limit-stats` endpoint exposes per-key monitoring stats. 30 new tests cover the rate limiter service and middleware behaviour.

## Implementation Summary
`ProcessChunkJob` was updated to support the Letter channel DOCX rendering pipeline. When the channel is Letter, the job now reads the DOCX template bytes once from `ITemplateBodyStore` (path stored in `TemplateSnapshot.ResolvedHtmlBody`), then calls `IDocxTemplateRenderer.RenderAsync` once per recipient with extracted scalar values, and sets `DispatchRequest.BinaryContent` with the rendered bytes instead of `Content`. `LetterDispatcher.SendAsync` is called atomically once per recipient — no batch accumulation. Email and SMS channels continue to use `ITemplateRenderer` + `Content` as before. There was no pre-existing PDF accumulation code to remove in the current codebase. Integration tests (12 cases) cover: store read once per chunk, renderer called once/N times per recipient, BinaryContent populated + Content null, SendAsync called once per recipient, scalar extraction, error cases (empty body path, store read failure), and non-regression for Email channel.

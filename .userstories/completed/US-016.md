**As a** Claire (Template Designer)
**I want** `{{ if condition_key }}...{{ end }}` to show/hide paragraphs or table rows
**So that** I can conditionally display sections based on recipient data

**Specification context:**
> F-304: `{{ if condition_key }}` and `{{ end }}` markers each occupy a dedicated paragraph or table row. Boolean value determines visibility. No `{{ else }}` support. Nested `{{ if }}` blocks throw `TemplateRenderException` with message "Nested {{ if }} blocks are not supported (found inside '{{ if condition_key }}')."

**Business rules:**
1. Conditional keys are case-sensitive
2. Values must be boolean (true/false) — non-boolean values treated as false
3. Nested conditionals explicitly not supported

### Key design decisions
- Linear scan of block-level elements (paragraphs + table rows) from `CollectBlocks()` — flattens the document into a simple sequence for marker detection
- Uses `Regex` patterns for `{{ if key }}` and `{{ end }}` detection (spaces inside braces optional, consistent with F-301/F-302 patterns)
- Nesting detected by incrementing `depth` counter when inner `{{ if }}` is found before `{{ end }}`; throws `TemplateRenderException` immediately with outer key in message
- Missing condition keys treated as `false` per Business Rule 2
- Markers removed in all cases; content removed only when condition is false
- Only main document body processed (not headers/footers, per F-304 spec)

### Tests (15 passing)
- True condition: content kept, markers removed (paragraphs and table rows)
- True condition: surrounding content preserved, multiple content paragraphs kept
- False condition: content and markers removed (paragraphs and table rows)
- False condition: surrounding content preserved, multiple sequential blocks evaluated independently
- Missing key treated as false
- No conditional markers: document unchanged
- Nested `{{ if }}`: throws `TemplateRenderException` with outer key in message
- Guard clauses: null document / null conditions throw `ArgumentNullException`

### Notes:

- The domain AST classes (FilterExpression, LeafFilterExpression, CompositeFilterExpression) are in `CampaignEngine.Domain.Filters` namespace and use System.Text.Json polymorphic serialization via a custom converter.
- All filter values are parameterized via Dapper DynamicParameters — no string interpolation of values occurs anywhere in the translation path.
- The FilterExpressionDto in the Application layer (pre-existing) is preserved and used by the SqlServerConnector for backwards compatibility. The new domain AST translates to FilterExpressionDto before passing to the connector.
- The FilterBuilder Razor page is accessible to Operator and Admin roles. It uses a JavaScript AST builder that communicates with the preview API via fetch().
- 44 unit tests cover: all 8 comparison operators, IS NULL/NOT NULL, IN with expansion, empty IN sentinel, injection prevention, AND/OR nesting, depth limit, JSON round-trip, validator acceptance and rejection cases, and relative date resolution.

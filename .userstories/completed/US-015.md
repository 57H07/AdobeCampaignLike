**As a** Claire (Template Designer)
**I want** to render a collection as repeated table rows
**So that** I can display invoice line items, order details, etc.

**Specification context:**
> F-303: Marker row `{{ collection_key }}`, template row with `{{ item.field }}`, end row `{{ end }}`. Engine duplicates template row per item. Missing `{{ end }}` throws `TemplateRenderException` with message "Missing {{ end }} for collection '{key}'."

**Acceptance criteria:**
- [x] Marker row `{{ collection_key }}` identifies start of collection
- [x] Template row contains `{{ item.field }}` placeholders
- [x] End row `{{ end }}` marks end of collection block
- [x] Engine duplicates template row once per item in collection
- [x] Marker and end rows are removed from output
- [x] Missing `{{ end }}` throws `TemplateRenderException` with clear message
- [x] Empty collections result in no rows (marker/end rows removed)

3. Collection keys and item field names are case-sensitive

## Implementation summary

### Files created
- `src/CampaignEngine.Infrastructure/Rendering/DocxTableCollectionRenderer.cs`
- `tests/CampaignEngine.Infrastructure.Tests/Rendering/DocxTableCollectionRendererTests.cs`

### Key implementation decisions
- Used `TableRow.CloneNode(deep: true)` for row duplication as per implementation notes
- `{{ item.field }}` replacement via regex on all `<w:t>` Text nodes in the cloned row
- Operations collected in forward pass and applied in reverse order to keep indices stable
- Missing collection keys treated as empty list (no rows, markers removed)
- Case-sensitive matching for both collection keys and item field names
- 14 unit tests covering: 3/5 items, field replacement, row order, surrounding static rows, empty collection, missing key, missing `{{ end }}`, null guard clauses

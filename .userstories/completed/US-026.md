## Implementation Summary
A dedicated `AdminSeeder` service was added to the Infrastructure layer. It runs on every startup via `app.SeedAdminAsync()` in `Program.cs` (after `app.Build()`). The seeder first ensures all three application roles (Admin, Designer, Operator) exist, then checks if any Admin-role user already exists — if so it returns immediately (idempotent). If no admin exists it creates a user from the `DefaultAdmin` configuration section (username, email, password, displayName) and assigns the Admin role. A startup warning is logged if the credentials match the well-known defaults shipped in `appsettings.json`. Credentials are fully configurable via environment variable overrides following the standard ASP.NET Core configuration hierarchy.
